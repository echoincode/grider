# ETF解套回本网格策略 - 技术实现方案[⚠️ **暂不开放**]

## 一、功能概述

### 1.1 需求背景

大量投资者存在持仓套牢的情况（成本价 > 当前价格），当前系统只能从零开始建仓，无法利用现有持仓进行成本摊薄。解套回本网格策略旨在帮助用户在已有持仓基础上，通过非对称网格配置快速摊低成本。

### 1.2 策略核心原理

| 区域 | 步长 | 目的 |
|------|------|------|
| **下方区域**（当前价以下） | 小步长、密集 | 价格越低越频繁买入，快速摊低成本 |
| **中部区域**（当前价附近） | 正常步长 | 保持流动性，做差价 |
| **上方区域**（成本价附近） | 大步长、稀疏 | 价格接近成本时少卖，等待解套 |

### 1.3 与普通网格的差异

| 特性 | 普通网格 | 解套网格 |
|------|---------|---------|
| 初始状态 | 空仓 + 全部资金 | 已有持仓 + 新投入资金 |
| 平均成本 | 无需追踪 | 必须追踪（加权平均） |
| 网格步长 | 固定步长 | 非对称步长（区域不同） |
| 交易逻辑 | 买入→卖出循环 | 买入摊低成本 → 卖出解套 |
| 回本目标 | 无 | 用户设定目标回本天数，系统调整策略 |
| 回测功能 | ✅ 支持 | ❌ 不支持（见说明） |

### 1.4 关于回测功能的说明

**解套网格不提供回测功能**，原因如下：

1. **时间逻辑矛盾**：用户当前处于套牢状态（成本价 > 当前价），但回测是从历史数据开始模拟，历史某个时间点可能还没有套牢
2. **初始条件不匹配**：解套策略的核心是"已有持仓 + 新资金"，回测无法真实模拟用户在历史时刻的实际持仓
3. **参考价值有限**：强行回测的结果无法反映用户当前套牢状态下的真实效果

因此，解套模式下将**隐藏回测标签页**，仅提供策略分析功能。

## 二、当前实现的问题与局限性（⚠️ 重要）

### 2.1 回本概率计算的概念混淆

**问题**：当前用 Z-score 计算的是"价格在 N 天内到达成本价的概率"，这是基于**价格方向性运动**的假设。但网格收益的本质来自**价格波动**（低买高卖做差价），而非价格回归。

**影响**：
- 即使价格永远回不到成本价，如果波动足够大，网格交易依然能通过频繁买卖累积利润来回本
- 如果价格单边下跌，即使概率很高，实际也无法回本

### 2.2 步长调整的因果链不成立

**问题**："目标天数越短 → 缩小步长 → 提高交易频率"这个逻辑有问题：

| 调整 | 效果 | 副作用 |
|------|------|--------|
| 步长变小 | 交易频率提高 | 单笔利润变小 |
| 交易频率提高 | 累积收益可能增加 | 交易成本（手续费、滑点）增加 |
| 净效果 | **不确定** | 可能更差 |

### 2.3 "预期摊薄成本"与网格策略无关

**问题**：预期摊薄成本本质就是**加权平均成本**，只跟投入多少新资金有关：

```
摊薄后成本 = (原有持仓 × 原成本 + 新资金) / (原有持仓 + 新资金/现价)
```

无论你用什么策略，只要投入相同的新资金，摊薄后的成本都是一样的。网格策略无法改变这个数学事实。

### 2.4 改进方向

建议重新设计为**场景分析模式**：

| 场景 | 说明 | 计算内容 |
|------|------|---------|
| **价格横盘** | 仅靠网格利润回本 | 需要多少天 |
| **价格下跌10%** | 越套越深 | 需要追加多少资金才能回本 |
| **价格上涨10%** | 价格回归 | 最快回本时间 |

同时在 UI 上添加**风险提示**：
> 解套策略本质是**追加资金摊低成本**，并非无风险策略。如果标的持续下跌，可能面临**越套越深**的风险。

## 六、策略分析指标

### 6.1 解套网格专属指标

| 指标 | 说明 | 计算方式 |
|------|------|---------|
| **成本降低幅度** | 预期平均成本下降比例 | 基于网格配置和资金分配估算 |
| **预期回本时间** | 用户设定的目标回本天数 | 用户输入（1-730天），系统据此调整策略 |
| **回本概率** | 在目标时间内回本的概率 | ⚠️ 存在概念混淆，需重新设计 |
| **最大预期浮亏** | 可接受的最大额外亏损 | 根据新资金和网格配置计算 |

### 6.2 与普通网格共用指标

| 指标 | 说明 |
|------|------|
| 适宜度评分 | 标的适合网格交易的程度 |
| 网格配置 | 步长、数量、价格水平 |
| 资金分配 | 底仓、网格交易、预留资金 |
| ATR比率 | 当前波动率水平 |

### 6.3 场景分析指标（待实现）

| 场景 | 指标 | 计算方式 |
|------|------|---------|
| **价格横盘** | 仅靠网格利润回本天数 | 网格利润 / 每日预期收益 |
| **价格下跌10%** | 需要追加的资金 | 目标成本 × 总持仓 - 当前资金 |
| **价格上涨10%** | 最快回本时间 | 价格回归到成本价所需时间 |

## 八、核心设计原则

1. **策略隔离**：普通网格和回本网格使用独立的分析入口和计算逻辑
2. **渐进式部署**：先完成后端，再开发前端，可独立测试
3. **配置化**：回本模式的区域划分比例可配置
4. **向后兼容**：所有新增方法不修改现有方法签名
5. **无回测设计**：解套模式下隐藏回测功能，避免用户误解
6. **目标驱动**：用户设定回本天数目标，系统自动调整策略参数
7. **诚实透明**：明确标注计算的局限性和风险

## 十一、暂不开放说明

### 11.1 当前状态

解套回本网格策略的代码实现已完成，但由于以下原因**暂不向用户开放**：

1. **回本概率计算存在概念混淆**：当前计算的是价格方向性回归的概率，而非网格交易通过波动获利回本的概率
2. **步长调整逻辑不严谨**：缩小步长增加交易频率的同时会降低单笔利润，净效果不确定
3. **缺乏场景分析**：没有提供不同市场情景下的回本路径分析
4. **风险提示不足**：没有明确告知用户解套策略的本质风险

### 11.2 下一步计划

1. **重新设计回本概率计算**：基于网格交易的实际收益模型，而非价格方向性概率
2. **添加场景分析**：计算价格横盘、下跌、上涨三种场景下的回本路径
3. **增强风险提示**：在 UI 上明确标注策略的风险和局限性
4. **验证计算逻辑**：通过回测验证网格收益模型的合理性

### 11.3 重新开放条件
